一、定位：從單模型協作進化到多模型閉環系統

舊版的協作框架假設「一個模型處理所有 AI 工作」。
新版 2.0 承認現實：不同智能工具擅長不同層級的思考。

因此，本框架由四個角色組成：

  • 抽象者（The Abstractor）： 負責推理、方法論、結構化、定義問題。
  • 技術者（The Technician）： 負責程式、技術落地、資料操作。
  • 整合者（The Synthesizer）： 負責將資料、任務、格式轉換為可跨系統使用的記憶體（JSON/CSV）。
  • 人類決策者（Human Arbiter）： 負責意圖、價值、裁決、方向、倫理。

四者共同形成一個「多層、多格式、多迭代」的閉環系統，用於高效率建構複雜產品、流程、資料流與方法論。

二、四大角色與職能

1. 抽象者（The Abstractor）——推理與結構層

  • 定義問題、目的、邊界。
  • 建立方法論、流程、架構。
  • 將自然語言轉換成可被其他角色理解的「結構化任務」。
  • 為整個系統提供「方向」與「解釋層」。

2. 技術者（The Technician）——構築與驗證層

  • 撰寫程式碼、修正錯誤、調校前後端。
  • 提供可執行的技術成果（功能、介面、API、資料）。
  • 產出可用於跨系統的結構化輸出（如 CSV）。
  • 驗證可行性與落地成本。

3. 整合者（The Synthesizer）——記憶與資料層

  • 管理 JSON（邏輯記憶架構）。
  • 管理 CSV（可分析、可視覺化的資料表）。
  • 負責 Notion、 Excel、語意記錄的雙向同步。
  • 確保整個系統的「狀態永遠一致」。

4. 人類決策者（Human Arbiter）——意義與價值層

  • 決定方向與優先順序。
  • 判斷價值，而不是資料。
  • 為所有模型提供意圖（Intention）。
  • 負責倫理、限制、成果、承擔風險。

三、閉環循環：Capture → Structure → Build → Record → Aggregate → Decide

  1. Capture（捕捉）： 人類輸入自然語言意圖。
  2. Structure（結構化）： 抽象者將意圖轉換成格式化任務與方法。
  3. Build（構築）： 技術者依據任務產生程式與技術成果。
  4. Record（紀錄）： 整合者將成果整理成 JSON / CSV。
  5. Aggregate（聚合）： 抽象者分析記錄，抽出模式與洞察。
  6. Decide（裁決）： 人類最終決策並啟動下一輪。

這個閉環形成一個自我增強、越用越強的個人作業系統。

四、方法論基礎：從單工具工作者進化為「一人多部門」系統

基礎 內涵說明
分層思考 不同層級的問題由不同智能負責，使系統不再混亂。
多格式記憶體 JSON 管邏輯、CSV 管資料、自然語言管意圖。
閉環優化 每次迭代都會讓下一輪更準確、更快。
人類裁決 意義、倫理、風險永遠回到人類，而不是模型。
