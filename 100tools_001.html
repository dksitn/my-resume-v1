🗺️ 產品綜述：夜間安全熱點圖 (S01)

這是我們「助人 × 自利開放地圖平台」的首個 MVP 站型。S01 旨在透過群眾回報 (Crowdsourcing) 蒐集夜間不安全事件，並經過嚴格的社群驗證 (Governance) 後，在地圖上呈現為可信賴的熱點圖，協助夜歸通勤者、旅人或家長避險。

🛡️ 技術骨架與安全保證 (R3/R5)

本站型建立在 Supabase (PostgreSQL + RLS) 的穩固架構之上，保證數據的**安全**與**可擴展性**。

  • 資料安全 (R5 RLS): 所有提報數據均受到 Row Level Security 嚴格保護。只有經過審核 (Status = 'verified') 的報告，才允許公開讀取。
  • 高權限隔離 (R2): 後台的審核和信譽分數計算，全部透過 Edge Functions 隔離運行，絕不暴露超級金鑰 (service_role_key) 於前端。
  • 極致性能 (R3): 採用 Next.js (SSG/ISR) 架構，確保熱點圖的快速載入，同時滿足 Google Core Web Vitals (SEO) 規範。

📊 治理與數據品質 (R1)

S01 透過數據驅動的 信譽系統，解決群眾外包最擔心的「惡作劇」和「誤報」問題。

  1. **灰層 (Grey Status):** 所有新的報告默認為 'grey' 狀態，不公開顯示。
  2. **信譽權重:** 只有擁有足夠 reputation 分數的貢獻者提交的報告，才能更快進入審核流程。
  3. **變現反哺:** 未來將透過 B2B API 收益，反哺高信譽的貢獻者，形成正向循環。

🗺️ 助人 × 自利開放地圖平台 (Help-and-Earn Open Map Platform)

GitHub 標籤 (Topics) 建議： nextjs supabase postgresql gis programmatic-seo community-governance maplibre monetization

一個以「數據治理」和「Programmatic SEO」為核心，旨在量產 100 種「助人 × 自利」站型的通用資料核心。我們將社群的力量結構化，轉化為具備商業價值與社會價值的開放數據服務。

🌟 核心價值與產品定位 (R1 One-Pager)

1. 核心哲學：助人 × 自利

我們建立一個可信賴的通用資料核心，用於承載所有基於地理位置或社群回報的數據（Place, Report, Resource）。

  • 價值鏈： 低摩擦供稿 → 社群驗證 → 結構化資料 → Programmatic SEO → B2C/B2B 變現 → 反哺供稿者。
  • 北極星指標 (North Star Metric): verified_reports_per_1k_MAU (每千名月活躍用戶產生的已驗證報告數量)。

2. 商業模式 (Monetization)

平台採用多維度變現策略，確保永續營運。

  • 消費者 (B2C): Pro alerts, Offline Map, Advanced Filters, CSV Export.
  • 企業用戶 (B2B - SaaS): District Safety API, Recruiting Dashboard, Trend Dashboard (高價值、高權限數據服務)。
  • 贊助/聯盟 (Sponsor/Affiliate): Sponsored Pin, Brand Section, Whitelist Ads.

🛠️ 生產級技術架構 (R3 系統思維)

我們的架構採用 SaaS 級別的解耦式微服務，專為性能和 Programmatic SEO 而優化。

服務類別 技術棧 (Tech Stack) 關鍵優勢 (R3 心法)
前端 & SEO Next.js (SSG/ISR) on Vercel 極致的 LCP/CLS 性能優化，實現大規模的 Programmatic SEO 頁面生成。
資料庫 (SSOT) Supabase (PostgreSQL + RLS) 關係型數據模型支持複雜的 治理邏輯 和 PostGIS 地理空間查詢。是所有社群驗證和信譽系統的唯一真相來源。
高權限/自動化 Cloudflare/Netlify Edge Functions 承載高權限 API 呼叫、異步同步 (例如：解決 Supabase 免費版休眠問題)。
搜尋引擎 Meilisearch 提供高速、多維度、即時的全文檢索服務。
地圖渲染 Leaflet / MapLibre 輕量且開源的地圖渲染解決方案。

🛡️ V1.0 衝刺：核心架構與安全實踐

本次衝刺的目標是建立一個可上線、安全的最小可愛產品 (MLP) 基礎。

1. 核心功能範疇 (Selected Features)

我們將聚焦於最具社群驗證潛力與地圖剛需的 10 個站型，其中第一個 MVP 將會是：

ID 名稱 實體 簡述
S01 夜間安全熱點圖 Report 蒐集夜間不安全事件的群眾回報，提供避險與警示。
V05 插座/Wi-Fi 友善咖啡 Place 標記可久坐工作的咖啡館，是數位遊牧的核心需求。
C09 產業薪資透明榜 Resource 匿名提交，聚合各產業薪資區間，實現數據透明化。
... (總計 10 個核心功能)

2. 生產級安全重點 (R2/R5)

  • RLS 實施 (核心)： 所有資料表（Place, Report, User）必須啟用 Supabase 的 Row Level Security 政策。確保使用者只能讀取、寫入或修改自己有權限的資料。
  • 金鑰隔離： 嚴格遵循最小權限原則，將 SUPABASE_SERVICE_ROLE_KEY (高權限金鑰) 僅存放在 Edge Functions 的環境變數中，絕不暴露於前端。
  • 廣告法規 (R2)： 所有付費連結 (Sponsored / Affiliate) 必須包含 rel="sponsored" 屬性，並在介面上進行清晰的法規揭露。
