資訊專案成就 (仰賴與Google Gemini溝通開發，具備PM能力)

核心成就：全端 API 整合 (Railway 平台)

  • [全端 API 整合] 成功串接前端 `fetch` 請求 與 n8n 後端 API，實現從 Railway PostgreSQL 資料庫動態載入商品資料，並渲染至 `products.html` 頁面。
  • [全端偵錯 (RCA)] 獨立診斷並修復前端 `TypeError: products.forEach is not a function` 關鍵錯誤。
  • [根本原因分析] 透過瀏覽器 F12 開發者工具 與 API 直接呼叫測試，成功追蹤 `TypeError` 的根本原因 (RCA) 為：n8n 後端 API 回應邏輯錯誤，回傳「單一物件」而非前端預期的「陣列」。
  • [後端修復] 成功修復 n8n 後端節點的回應邏輯 (將 `First Item` 改為 `All Items`)，解決 `TypeError`，完成 V3.0 動態資料串接。

後端、資料庫與平台維運 (Backend, DB & Operations)

  • [平台評估與遷移] 獨立診斷並驗證特定雲平台 (Zeabur) 的 n8n 與 PostgreSQL 服務間存在連線 Bug。
  • [架構遷移] 評估並決策，成功將完整專案 (包含 Node.js 前端、n8n 後端、PostgreSQL) 遷移至 Railway 平台，確保服務穩定性。
  • [DB 連線偵錯] 解決 Railway PostgreSQL 的 SSL 憑證與 n8n 自動偵測衝突問題，完成手動憑證配置並成功連線。
  • [資料庫設計] 為購物網站專案獨立設計並 `CREATE` (建立) `categories` 與 `products` 關聯式資料表。
  • [SQL 實務] 具備 SQL DML (Data Manipulation Language) 實務經驗 (`INSERT`, `SELECT`)，成功寫入並驗證 6 筆商品與 10 筆類別資料。
  • [SQL 偵錯] 具備 SQL 語法偵錯能力 (如修正 `TIMESTAMPTZ` 拼寫錯誤)，並能獨立排除 `relation already exists` 等資料庫狀態錯誤。
